Tại sao cần data driver-level?
Hiện tại: Aggregate only
Chỉ biết tổng FT tháng này giảm 300 — không biết 300 đó là FT churn thực sự hay FT downgrade xuống PT vì giờ hoạt động giảm xuống dưới 150h.
Cần: Cohort Retention
Theo dõi từng driver_id qua nhiều tháng → tính tỷ lệ còn active M+1, M+2, M+3... theo từng NIM cohort. Hiện tại không làm được.
Cần: Segment Transition
Biết driver A tháng 1 là FT → tháng 2 thành PT hay Inactive? Đây là key để phân biệt downgrade vs churn — action khác nhau hoàn toàn.
Cần: Earnings Breakdown
Phân biệt organic earnings vs incentive để biết Return driver quay lại vì platform thực sự tốt hay chỉ vì campaign — tránh đốt budget vô ích.
Table 1 — driver_monthly_activity (BẮT BUỘC)
driver_monthly_activity
1 row = 1 driver × 1 tháng. Bao gồm cả tháng inactive (is_active = 0) để tính retention.
Primary Key: driver_id + period
FieldTypeExampleDùng để phân tích gì
driver_id MUST STRING "DRV00001" Key để track từng driver qua nhiều tháng. Không có field này thì không làm được bất kỳ cohort analysis nào.
city_id MUST STRING "HAN" Phân tích riêng HAN vs SGN.
period MUST DATE "2025-01-01" Grain tháng — mỗi row là 1 tháng của 1 driver.
is_active MUST INT (0/1) 1 Quan trọng nhất cho retention. Phải có cả row inactive (=0) để biết driver nào đã rời. Nếu chỉ có row active thì không tính được churn.
segment MUST STRING "FT" / "PT" / "NIM" / "NLM" / "Return" Segment của driver trong tháng đó. Tính segment transition matrix: FT→PT, PT→Inactive, NLM→FT, v.v.
active_hours MUST FLOAT 156.5 Giải quyết câu hỏi FT churn vs downgrade. Nếu driver từng là FT (>150h) mà tháng sau active_hours = 80h → downgrade sang PT. Nếu is_active=0 → churn thực sự.
completed_orders MUST INT 342 Volume đơn hoàn thành. Tính EPO (earnings per order), productivity per hour.
completed_stoppoints MUST INT 487 Đã có trong data aggregate. Verify vs data hiện tại.
gross_earnings MUST INT (VND) 4500000 Tổng thu nhập. Tính EPH (earnings per hour) — metric quan trọng nhất để hiểu vì sao driver rời.
incentive_earned MUST INT (VND) 800000 Phân biệt organic vs incentive-driven. Return driver có incentive_earned cao → họ quay lại vì campaign. organic_earnings = gross - incentive.
service_type_primary NICE STRING "1H" Service chính driver làm nhiều nhất. Phân tích FT/PT breakdown theo service.
online_days NICE INT 22 Số ngày online trong tháng. Phân biệt driver online 30 ngày × 5h vs 15 ngày × 10h.
acceptance_rate NICE FLOAT (%) 87.3 Tỷ lệ nhận đơn. FT giảm AR → dấu hiệu chán nản trước khi churn.
Table 2 — driver_master (BẮT BUỘC)
driver_master
1 row = 1 driver. Thông tin static — dùng để xác định NIM cohort và nguồn tuyển dụng.
Primary Key: driver_id
FieldTypeExampleDùng để phân tích gì
driver_id MUST STRING "DRV00001" Join key với driver_monthly_activity.
city_id MUST STRING "HAN" Thành phố đăng ký.
first_active_month MUST DATE "2025-01-01" Xác định NIM cohort. Đây là tháng driver được tính là NIM. Mọi cohort retention chart đều group theo field này.

Công thức Retention — với data trên sẽ tính được
1. Cohort Retention Rate (M+N)
-- NIM cohort Jan-2025:
cohort = drivers WHERE
  first_active_month = '2025-01'

M+1 retention =
  COUNT(active in Feb-25)
  ÷ COUNT(cohort) × 100

M+2 retention =
  COUNT(active in Mar-25)
  ÷ COUNT(cohort) × 100
→ Vẽ được retention curve cho mỗi NIM cohort. So sánh Jan-25 vs Jun-25 vs Jan-26 cohort để thấy chất lượng tuyển mới đang tốt lên hay xấu đi.
2. FT Churn vs Downgrade
-- Driver từng là FT tháng N:
prev_FT = segment[N] = 'FT'

Tháng N+1:
→ is_active=0 → CHURN
→ is_active=1
  AND active_hours < 150 → DOWNGRADE
  AND active_hours ≥ 150 → RETAIN FT
→ Biết được bao nhiêu % FT giảm là do churn thực (cần win-back) vs downgrade (cần giữ giờ online).
3. NIM Cohort Graduation Quality
-- Trong số NIM còn sống M+2:
graduated_FT =
  COUNT WHERE segment[M+2]='FT'
  ÷ COUNT(cohort) × 100

graduated_PT =
  COUNT WHERE segment[M+2]='PT'
  ÷ COUNT(cohort) × 100
→ Thay vì chỉ biết "NIM→NLM 81%", biết được bao nhiêu % trở thành FT (chất lượng cao) và bao nhiêu % chỉ là PT.
4. Return Driver Quality
-- Phân loại Return theo gốc:
Return FROM FT:
  prev_segment = 'FT'
  → có thể win-back về FT

Return FROM PT:
  prev_segment = 'PT'
  → kỳ vọng thấp hơn

Compare EPH sau khi return
→ Biết Return-ex-FT có EPH cao hơn Return-ex-PT không → ưu tiên activate đúng nhóm.
5. Incentive Dependency Index
IDI = incentive_earned
      ÷ gross_earnings × 100

-- So sánh theo segment:
FT IDI vs PT IDI vs Return IDI

-- Monitor over time:
IDI tăng → platform
phụ thuộc incentive hơn
→ Return driver có IDI cao → họ quay lại vì campaign, không phải vì platform tốt → cắt incentive sẽ churn lại.
6. Segment Transition Matrix
-- Mỗi cặp (segment[N], segment[N+1]):

FT → FT: retain rate
FT → PT: downgrade rate
FT → inactive: hard churn
PT → FT: upgrade rate
PT → inactive: soft churn
inactive → Return: reactivation
→ Ma trận chuyển hạng đầy đủ → thấy điểm rò rỉ lớn nhất ở đâu trong lifecycle.
