Sub-Agents — .claude/agents/

Mỗi file trong .claude/agents/ là một "chuyên gia con" với persona, hành vi và output format riêng. Claude tự động chọn đúng agent theo loại task — không cần gọi thủ công.

  • 🔍
    sql-analyst.md
    Senior Data Engineer — Viết SQL production-grade cho BigQuery. Kích hoạt khi cần query AR, FR, CPO, Active Drivers, EPH, RPH, Mini-hub metrics. Tư duy Kimball dimensional modeling, luôn thêm data validation layer.
  • 📝
    report-writer.md
    Report Writer — Có data sẵn → viết báo cáo Lark Docs chuẩn enterprise với cấu trúc 3 phần: Executive Summary / Analysis Framework / Value Realization.
  • 🏆
    competitive-intel.md
    Competitive Intelligence Analyst — Phân tích động thái Grab Express, Be Delivery, XanhSM, SPX/GHN. Đánh giá tác động lên supply tài xế và chiến lược phản ứng.
  • 📣
    driver-comms.md
    Driver Communications Writer — Viết thông báo, script, nội dung Zalo OA cho tài xế. Ngôn ngữ đơn giản, thực tiễn, phù hợp từng tầng tài xế (Part-time, Full-time, Mini-hub).
  • 🌐
    landing-page-builder.md
    Landing Page Builder — Tạo landing page HTML với brand Ahamove. Kích hoạt khi cần tuyển tài xế, campaign EV, Mega Sale. Dùng inline SVG, không dùng thư viện chart ngoài.
  • ✍️
    content-writer.md
    Content Writer — Caption mạng xã hội, blog, LinkedIn post, script video cho kênh cộng đồng tài xế Ahamove.
  • 🎪
    event-planner.md
    Event Planner — Lên kế hoạch event offline/online cho cộng đồng tài xế: run-of-show, checklist logistics, ngân sách, KPI đo lường sau event.
  • 📅
    meeting-prep.md
    Meeting Prep — Chuẩn bị trước cuộc họp quan trọng: tóm tắt context, câu hỏi cần hỏi, rủi ro cần lường trước, action items dự kiến.
  • 📊
    weekly-review.md
    Weekly Review — Tổng kết cuối tuần: so sánh KPI vs target, highlight top 3 rủi ro, action items tuần tới, gửi draft cho stakeholders.

🔄 Workflow: Memory hoạt động như thế nào?

Session 1 — Khanh dạy Claude
"Tôi là Khanh, Driver Management Leader. Đừng bao giờ bịa số liệu. Trả lời bằng tiếng Việt."
Claude ghi nhớ → tạo file
Claude tự tạo user_profile.md, user_rules.md, user_voice.md trong thư mục memory.
Session 2 — Khanh quay lại
Claude đọc lại memory files → đã biết Khanh là ai, không hỏi lại từ đầu.
Khanh feedback → Claude cập nhật
"Lần sau đừng dùng thư viện chart ngoài" → Claude update feedback_visualization.md.
Session N — Claude đã "quen" Khanh
Không cần giải thích lại context, Claude làm việc ngay từ đầu với đúng giọng văn và format.

📁 Workflow 4 — Tại sao cần cấu trúc output/Ahamove/ như vậy?

Không có cấu trúc → Claude lưu file tùy ý → sau 1 tháng không tìm được gì. Cấu trúc 7 folder theo chức năng (không theo nguồn) giúp:

  • Tìm file theo chủ đề trong <10 giây
  • Claude biết chính xác lưu gì ở đâu — không hỏi
  • README.md là "bản đồ" luôn cập nhật
  • Git history cho thấy evolution của từng chủ đề

Hiệu quả — Có cấu trúc vs Không có
Tình huống Không có cấu trúc file Có cấu trúc Pulu-workspace
Mở session mới Phải giới thiệu lại "Tôi là Khanh, làm ở Ahamove..." mất 5-10 phút Claude đọc memory files → sẵn sàng làm việc ngay
Yêu cầu viết thông báo cho tài xế Claude viết văn phong "AI robot", thiếu thực tiễn, phải sửa nhiều lần driver-comms agent kích hoạt → đúng giọng tài xế, đúng format ngay lần đầu
Cần query SQL Claude hỏi 5 câu về DB platform, granularity, filter trước khi viết sql-analyst agent biết Khanh dùng BigQuery, tự hỏi đúng 1-2 câu cần thiết
Làm xong, tắt máy Không commit → sáng hôm sau ở văn phòng không có file mới Hook tự push GitHub + sync Drive → máy nào cũng có bản mới nhất
Sửa lỗi "đừng bịa data" Phải nhắc mỗi lần "đừng bịa nhé" — Claude quên sau mỗi session Feedback được lưu vào memory file → không bao giờ quên
Tìm lại báo cáo tháng trước File nằm lung tung trên Desktop, không nhớ tên README.md là index đầy đủ → tìm trong <10 giây
Làm việc trên 2 máy Copy USB / email file cho mình — dễ nhầm phiên bản git pull ở máy mới → có đúng version mới nhất
