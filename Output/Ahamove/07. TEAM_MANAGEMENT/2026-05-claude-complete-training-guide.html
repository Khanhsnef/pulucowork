Phần 1
AI Là Gì?
Giải thích không cần kỹ thuật — ai cũng hiểu được

Hãy nghĩ AI (Trí tuệ nhân tạo) như một nhân viên đã đọc gần như toàn bộ internet — sách, bài báo, Wikipedia, diễn đàn, tài liệu kỹ thuật… — rồi học cách trả lời câu hỏi, viết văn bản, phân tích và lý luận dựa trên tất cả kiến thức đó.

Khác với phần mềm thông thường (bấm nút → làm đúng 1 việc cố định), AI hiểu ngôn ngữ tự nhiên — bạn nói chuyện bình thường với nó như nói chuyện với người.

Học từ hàng tỷ văn bản

Trong quá trình training, Claude đọc hàng tỷ câu văn và học cách dự đoán "từ tiếp theo có thể là gì". Không có não bộ thật sự — nhưng kết quả giống suy nghĩ con người đến ngạc nhiên.

🧠

Context Window

Context window = "bộ nhớ làm việc" trong 1 cuộc trò chuyện. Claude thấy toàn bộ lịch sử hội thoại từ đầu → trả lời dựa trên TẤT CẢ context đó.

⚠
Tắt tab → Claude quên hết. Mỗi conversation mới = trang giấy trắng.
🚫

Claude KHÔNG thể:

• Nhớ bạn sau khi đóng hội thoại (trừ khi có Memory)
• Truy cập internet real-time (trừ khi được cài tool)
• Tự lưu file vào máy bạn
• Tự hành động khi không có lệnh

claude.ai (Web) — Dành cho mọi người

Truy cập: Mở trình duyệt → claude.ai → đăng nhập. Không cần cài gì.

✓ Chat hỏi đáp ✓ Upload file PDF/Excel/Word ✓ Projects (workspace) ✓ Mọi thiết bị ✗ Không thao tác file trên máy ✗ Không tự động hóa
💡
Projects trong claude.ai: Tạo "không gian làm việc" riêng cho từng chủ đề — lưu System Prompt cố định + tài liệu nền (SOP, brand guide). Mọi hội thoại trong Project đều kế thừa context đó, không cần paste lại mỗi lần.
🖥️

Claude Desktop — App cài trên máy

Truy cập: Tải tại claude.ai/download → cài đặt → đăng nhập. Giao diện giống claude.ai nhưng là app riêng.

✓ Tất cả tính năng của Web ✓ Không bị lẫn với tab trình duyệt ✓ Hỗ trợ MCP — kết nối Gmail/Calendar/Drive ✗ Vẫn cần internet ✗ MCP phải tự cấu hình (phức tạp)
💡
MCP (Model Context Protocol): Chuẩn kết nối cho phép Claude "với tay" sang Gmail, Google Calendar, Notion, Slack... Desktop hỗ trợ cài MCP thủ công — nhưng cần kỹ thuật. Pulu-workspace đã cài sẵn và cấu hình tự động.
💻

Claude Code — Thao tác file thật trên máy

Phiên bản Claude chạy trong terminal (cửa sổ dòng lệnh). Không phải chat — Claude thực sự đọc, ghi, sửa file trên máy tính của bạn.

Pulu-workspace — Tại Sao Tiện Hơn Cho Daily Work?
So sánh cụ thể từng tình huống thực tế
1
Claude không bao giờ hỏi "bạn làm gì?" lần thứ hai
Không có Pulu-workspace: "Xin chào, tôi là Khanh, Driver Management Leader tại Ahamove. Tôi phụ trách xe máy Instant, không phụ trách xe tải…" (lặp lại mỗi session)
Có Pulu-workspace: Memory files → Claude biết bạn là ai, scope là gì, metrics cần theo dõi là gì → làm việc ngay từ câu đầu tiên
2
Không mất dữ liệu dù quên commit
Không có Pulu-workspace: Làm xong quên push → sáng hôm sau ở văn phòng không có file mới, phải làm lại
Có Pulu-workspace: settings.json hook → mỗi khi Claude dừng, tự động git push + sync Google Drive → máy nào cũng có bản mới nhất
3
Đúng chuyên gia đúng việc — không cần hướng dẫn
Không có Pulu-workspace: "Tôi muốn bạn đóng vai SQL analyst và viết query theo chuẩn BigQuery với data validation..." (paste mỗi lần)
Có Pulu-workspace: Claude tự nhận ra "cần query data" → kích hoạt sql-analyst agent → viết đúng format BigQuery ngay
4
Lệnh tắt /skill thay vì giải thích dài
Không có Pulu-workspace: "Tạo cho tôi 1 file Excel chuyên nghiệp với các sheet sau, format bảng đẹp, dùng màu brand của Ahamove là #0E4174…" (20 dòng instruction)
Có Pulu-workspace: /xlsx → Claude biết chính xác cần làm gì, output chuẩn ngay lần đầu
5
Claude học từ feedback của bạn vĩnh viễn
Không có Pulu-workspace: "Lần sau đừng dùng thư viện chart ngoài nhé" → Claude quên sau session đó → lần sau phải nhắc lại
Có Pulu-workspace: feedback được lưu vào memory/feedback_visualization.md → không bao giờ quên, không cần nhắc lại
6
Thao tác file thật — không copy-paste thủ công
Không có Pulu-workspace: Copy data từ Excel → paste vào chat → Claude phân tích → copy kết quả → paste vào file
Có Pulu-workspace: Claude đọc thẳng file Excel trên máy → xử lý → ghi kết quả ra file mới → mở file lên là xong

IDE Là Gì? Tại Sao Dùng IDE + Claude?
Giải thích cho người chưa biết gì về lập trình — tại sao nhiều người dùng VSCode/Cursor để làm việc với Claude

IDE là gì?

IDE (Integrated Development Environment) = Môi trường phát triển tích hợp — một phần mềm soạn thảo văn bản/code siêu mạnh, giống như Word nhưng dành cho viết code và quản lý file.

Hãy hình dung IDE như một "văn phòng kỹ thuật số" đầy đủ tiện nghi: bạn có thể xem cây thư mục file bên trái, soạn thảo bên phải, chạy terminal phía dưới, và Claude ngồi ngay bên cạnh — tất cả trong 1 cửa sổ.

💡
IDE phổ biến nhất hiện nay:
• VS Code (Microsoft) — miễn phí, phổ biến nhất
• Cursor — VS Code nhưng tích hợp AI sâu hơn
• Windsurf — IDE AI-first mới nổi
• JetBrains (IntelliJ, PyCharm) — cho developer chuyên nghiệp

Phần 13
Từ Điển Thuật Ngữ
Mọi thuật ngữ bạn sẽ gặp khi làm việc với Claude
Thuật ngữGiải thích đơn giản
AITrí tuệ nhân tạo — phần mềm có khả năng xử lý ngôn ngữ và lý luận
LLM"Large Language Model" — loại AI được train trên lượng văn bản khổng lồ. Claude và ChatGPT đều là LLM
PromptCâu lệnh/câu hỏi bạn gửi cho Claude. Prompt tốt = kết quả tốt
Context Window"Bộ nhớ làm việc" — lượng text Claude xử lý được trong 1 hội thoại. Tắt tab → mất hết
HallucinationKhi AI tự bịa ra thông tin sai nhưng trình bày tự tin như thật. Luôn verify thông tin quan trọng
TrainingQuá trình "học" của AI — đọc hàng tỷ tài liệu để hiểu ngôn ngữ và cách trả lời
Knowledge CutoffNgày Claude ngừng được cập nhật kiến thức. Sau đó không biết tin tức mới
ModelPhiên bản AI cụ thể. Claude có: Haiku (nhanh/nhẹ), Sonnet (cân bằng), Opus (mạnh nhất)
TokenĐơn vị đo văn bản (~3/4 từ tiếng Anh). Giới hạn context window tính bằng token
APICách kết nối Claude vào phần mềm/app khác. Developer dùng API để build ứng dụng có AI
AgentClaude được trang bị thêm tools (tìm kiếm web, đọc file, chạy code) để tự thực hiện task phức tạp hơn
System PromptLệnh nền cài sẵn định hình cách Claude phản hồi. Bạn thường không thấy nhưng nó ảnh hưởng mọi câu trả lời
MCP"Model Context Protocol" — chuẩn kết nối Claude với Gmail, Calendar, Notion, Slack...
Projects (Workspace)Không gian làm việc trong claude.ai — lưu system prompt + tài liệu nền để dùng lại nhiều lần
Claude CodeClaude chạy trong terminal/IDE — có thể đọc/ghi file thật trên máy tính, thao tác codebase
Claude DesktopApp Mac/Windows giống claude.ai nhưng hỗ trợ kết nối MCP thêm tool ngoài
Pulu-workspaceMôi trường làm việc đầy đủ xây trên Claude Code — có Memory, Skills, sub-agents, auto-sync
MemoryHệ thống lưu trữ thông tin qua nhiều session — Claude nhớ bạn là ai, preferences, dự án đang chạy
SkillsLệnh tắt /tên trong Pulu-workspace — gõ /xlsx, /pptx, /pdf... Claude thực hiện task phức tạp theo quy trình định sẵn
Sub-agentClaude "phân thân" thành chuyên gia con — sql-analyst, report-writer, driver-comms... mỗi agent có persona và output format riêng
HookHành động tự động được kích hoạt khi có sự kiện — ví dụ: hook Stop = tự git push mỗi khi Claude dừng
IDE"Integrated Development Environment" — phần mềm soạn thảo code siêu mạnh (VSCode, Cursor). Khi tích hợp Claude, có thể thao tác file trực tiếp
VSCodeVisual Studio Code — IDE miễn phí phổ biến nhất của Microsoft. Cài Claude Code extension để dùng AI trực tiếp trong editor
CursorIDE AI-first xây trên nền VSCode, tích hợp Claude/GPT sâu hơn. Phổ biến trong developer 2024-2026
Diff ViewChức năng trong IDE hiển thị chính xác những gì thay đổi (màu đỏ = xóa, màu xanh = thêm) — kiểm soát được Claude đã làm gì
CLAUDE.mdFile cấu hình gốc trong Pulu-workspace — Claude đọc đầu tiên để hiểu context công ty, brand, quy tắc output
AnthropicCông ty tạo ra Claude. Thành lập 2021 tại Mỹ, tập trung vào AI an toàn
