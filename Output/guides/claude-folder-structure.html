Flow đọc file — Claude Code load gì, khi nào, theo thứ tự nào?

Mỗi khi bạn mở Claude Code, đây là trình tự chính xác các file được nạp vào context

A — Tự động khi khởi động phiên (Session Start)
1
⚙️ Global Settings
~/.claude/settings.json
Quyền thực thi, effortLevel, cài đặt model được nạp đầu tiên. Áp dụng cho mọi project, mọi folder. Không có project CLAUDE.md nào ghi đè được file này.
2
🔌 MCP Tools
~/.claude/.mcp.json
Danh sách MCP servers (NotebookLM, Gmail, GDrive…) được đăng ký. Claude biết công cụ nào tồn tại nhưng chưa kết nối — chỉ kết nối khi bạn dùng lần đầu.
3
📄 Project Rules
CLAUDE.md & AGENTS.md (project root)
File quan trọng nhất. Claude đọc CLAUDE.md của thư mục hiện tại, rồi leo lên các thư mục cha cho đến root. AGENTS.md nạp persona và template chuẩn. Hai file này override mọi default behavior của Claude.
4
⚙️ Project Hooks
.claude/settings.json (project)
Hooks được đăng ký: Stop hook sẽ tự động git commit + rsync sang Google Drive mỗi khi Claude kết thúc phản hồi. Project settings ghi đè global settings nếu có conflict.
5
💾 Memory Index
projects/…/memory/MEMORY.md
Claude đọc MEMORY.md (index) trước, sau đó chọn có chọn lọc các memory files liên quan đến câu hỏi đang xử lý. Không nạp toàn bộ — chỉ nạp những file phù hợp ngữ cảnh.
user_profile.md user_rules.md user_voice.md …
✓
🟢 Ready
Claude Code sẵn sàng
Toàn bộ context đã nạp. Claude đã biết: bạn là ai, làm gì, muốn output thế nào, có công cụ gì, và có hook nào chạy tự động.
B — Nạp theo yêu cầu (On-demand)
/ skill-name
Skills
Khi gọi /xlsx, /pdf, /pptx… Claude mới đọc skills/[name]/SKILL.md. Không dùng = không tốn context. Skill xong là giải phóng.
Task Agent
Sub-agents
Khi Claude spawn agent (sql-analyst, report-writer…), agents/[name].md trở thành system prompt của agent đó trong cửa sổ context riêng — không ảnh hưởng luồng chính.
First call
MCP Servers
Kết nối MCP (NotebookLM, Gmail…) chỉ mở khi bạn dùng lần đầu trong phiên. Nếu server cần auth, Claude sẽ hỏi bạn xác thực.

Claude quyết định viết vào đâu — và làm sao nhận biết?

Câu hỏi thực tế: tôi nhờ Claude "thêm permission", "nhớ điều này", "tạo agent" — nó sẽ sửa file nào?

🧭
Quy tắc vàng: Claude viết vào nơi file ĐÃ TỒN TẠI gần nhất
Nếu Cowork/.claude/settings.json tồn tại → Claude sửa file đó.  |  Nếu không có → Leo lên ~/.claude/settings.json.  |  Ngoại lệ duy nhất: Memory luôn ghi vào global, Skills luôn nằm ở global.
Khi bạn nói… → Claude làm gì và viết vào đâu
Bạn yêu cầu Claude viết vào file Môi trường Lý do
"Thêm permission cho npm" .claude/settings.json Project Project settings.json tồn tại → viết vào đó
"Nhớ điều này cho các lần sau" ~/.claude/projects/…/memory/ Global Memory luôn ghi vào global, dù đang ở project
"Sửa CLAUDE.md, thêm rule này" Cowork/CLAUDE.md Project CLAUDE.md trong thư mục hiện tại là mặc định
"Tạo agent sql-analyst" .claude/agents/sql-analyst.md Project Đang trong project → agent vào project agents/
"Thêm MCP server Notion" ~/.claude/.mcp.json Global MCP luôn global — dùng được mọi project
"Cài skill /okr" ~/.claude/skills/okr/ Global Skills luôn global — gọi được từ mọi nơi
"Tạo agent cho TẤT CẢ project" ~/.claude/agents/name.md Global Từ khoá "tất cả / mọi project / globally" → global
"Thêm note này, đừng commit" CLAUDE.local.md Local Từ khoá "đừng commit / riêng tư / local only"
"Thêm hook Stop tự động" .claude/settings.json Project Hook thường gắn với project workflow
Làm sao nhận biết Claude đang sửa file nào?
👁 Nhìn vào tool call trong UI
Mỗi khi Claude đọc hoặc viết file, UI hiển thị tên tool và đường dẫn. Đây là cách chính xác nhất:
Write /Users/…/.claude/settings.json ← Global
Edit Cowork/.claude/settings.json ← Project
Write Cowork/CLAUDE.local.md ← Local
Write /Users/…/memory/user_x.md ← Global memory
🔍 Đọc prefix đường dẫn
Global /Users/[you]/.claude/…
Project Cowork/.claude/… hoặc Cowork/CLAUDE.md
Local *.local.md hoặc *.local.json
Memory /Users/[you]/.claude/projects/…/memory/
Tip: Nếu không chắc, hỏi thẳng Claude: "Bạn vừa sửa file nào?" — Claude sẽ báo chính xác đường dẫn.
